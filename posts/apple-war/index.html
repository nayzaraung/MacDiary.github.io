Apple VS FBI

February 26 2016
news

ဒီဇင်ဘာလ ၂ ရက် ၂၀၁၅။         California ပြည်နယ် San Bernardino မြို့လေးရဲ့ ပါတီပွဲ တစ်ခုမှာ သေနတ်သမားနှစ်ဦးရဲ့ အကြမ်းဖက် ပစ်ခတ် တိုက်ခတ်မှုကြောင့် လူ ၁၄ ဦး သေဆုံးခဲ့ပြီး လူ ၂၀ ကျော် ထိခိုက်ဒဏ်ရာ ရရှိခဲ့ပါတယ်။

အဆိုပါ သေနတ်သမား နှစ်ဦးဖြစ်တဲ့ Syed Rizwan Farook နဲ့ Tashfeen Malik တို့ဟာ ပါကစ္စတန်နွယ်ဖွားဇနီးမောင်နှံ ဖြစ်ပြီး၊ Farrok (၂၈ နှစ်) က အမေရိကန်မှာ မွေးဖွားကြီးပြင်းတဲ့ (American Citizen) ကျန်းမာရေးဌါနမှ ဝန်ထမ်း တစ်ဦးလည်းဖြစ်ပါတယ်။ Malik (၂၉ နှစ်) ကတော့ Farook နဲ့ လက်ထပ်ပြီးနောက်မှ၊ အမေရိကန်မှာ အမြဲတမ်းနေထိုင်ခွင့်ရသူ (Permanent Resident) အဖြစ် အမေရိကန်ကို ရောက်ရှိလာသူ ဖြစ်ပါတယ်။

အကြမ်းဖက်မှုဖြစ်ပွားရာ နေရာ (Inland Regional Center) ရဲ့ ခန်းမဆောင် တစ်ခုမှာ Farook အလုပ်လုပ်တဲ့ ကျန်းမာရေးဌါနက ဝန်းထမ်း (၈၀) လောက် တွေဆုံပြီး ပါတီပွဲလေး ကျင်းပနေစဉ် Farook တို့ စုံတွဲက ရုတ်တရက် အကြမ်းဖက် သေနတ်နဲ့ ပစ်ခတ်ခဲ့တာဖြစ်ပါတယ်။ ဗုံးဖောက်ခွဲဖို့ပါ စီးစဉ်ထားခဲ့သော်လည်း၊ ဇနီးမောင်နှံဟာ သေနတ်နဲ့ ပစ်ခတ်ခဲ့ပြီးနောက် အသင့်ဌါးရမ်းထားသော sport utility vehicle (SUV) နဲ့ ထွက်ပြေးခဲ့ပြီး၊ ရဲများက လိုက်လံ ပိတ်ဆို့ဖမ်းဆီးခဲ့ရာ လေးနာရီခန့် အကြာမှာ နှစ်ဦးစလုံးကို အသေပစ်ခတ် ဖမ်းဆီးရမိခဲ့ပါတယ်။

ယင်းဖြစ်ရပ်ကြောင့် ယနေ့ ကမ္ဘာထိပ်တန်း ကွန်ပျူတာကုမ္ပဏီ Apple နဲ့ US အစိုးရထိပ်တန်း အေဂျင်စီ FBI တို့ အကြား လုံခြုံရေးဆိုင်ရာ အငြင်းပွားမှု၊ လွန်ဆွဲမှုတွေ ဖြစ်ပေါ်ခဲ့ပြီး တရားရုံးမှာ ယှဉ်ပြိုင် ရင်နေဆိုင်နေကြရပါပြီ။

သေဆုံးသူတရားခံ Farook ရဲ့ iPhone 5C ထဲက encrypt (အချက်လက်များကို ကုဒ်အဖြစ်ပြောင်းလဲသောစနစ်) လုပ်ထားသော အချက်အလက်တွေကို FBI က ရယူဖို့ ကြိုးပမ်းပေမယ့် iPhone မှာ passcode ခံထားတဲ့ အတွက် ဖွင့်မရပဲ အခက်တွေ့နေပါတယ်။

အဆိုပါ passcode ကို ၁၀ ကြိမ် မှာယွင်းစွာ ရိုက်သွင်းပါက ဖုန်းအတွင်းရှိ Data များ အလိုအလျောက် ပျက်ပြယ်သွားမှာဖြစ်တဲ့အတွက် မှန်းဆ၍ မနှိပ်ရဲပေ။ ထို့ကြောင့် ကယ်လီဖိုးနီးယာ တရားသူကြီး “Sheri Pym” က Apple ကို Farook iPhone 5C ပွင့်ရန်အတွက် နည်းပညာအကူအညီပေးရန် အမိန့်ထုတ်ပြန်လိုက်သည်။

လက်ရှိမှာ ယခုလို passcode မသိသော ဖုန်းများအတွက် ကယ်ပေါက် (loophole) အနေနဲ့ Apple က passcode သိရန်မလိုပဲ firmware အသစ်ပြန်တင်ရန်သာ လုပ်ဆောင်ပေးထားပါတယ်။ ဒါကလည်း customers တွေရဲ့ iPhone ချို့ယွင်းတဲ့အခါ ပြန်လည် ပြင်နိုင်အောင် ဆိုတဲ့ ရည်ရွယ်ချက်နှင့်ပါ။

ဒါကြောင့်မို့ FBI က Farook ရဲ့ iPhone 5C ကို data များ မပျောက်ပျက်စေပဲ ဖွင့်နိုင်ရန်၊ အစိုးရအနေနဲ့ အထူးလုပ်ဆောင်ချက်တွေ ပါဝင်တဲ့ alternative OS (GovtOS) ကို ရေးပေးဖို့ တောင်းဆိုနေပါတယ်။ အဆိုပါ “GovtOS” တွင် passcode ကို အကြိမ်ပေါင်းများစွာ ထည့်သွင်းစမ်းနိုင်ရန် အကြိမ်ရေကန့်သတ်ထားသည့် time-delay စနစ်ကို မထည့်သွင်းရန်နှင့် passcode ကို လက်နဲ့ ရိုက်ထည့်ရန်မလိုပဲ FBI မှ script လေးတစ်ခု run လိုက်ရုံနဲ့ အလိုလျောက်ရှာဖွေ ပွင်းသွားစေရန် ပြုလုပ်ပေးဖို့ တောင်းဆိုနေပါတယ်။

လတ်တလောမှာ Apple က iCloud နဲ့ backup လုပ်ထားသော data များကို encrypt လုပ်ပေးထားပြီး၊ keys များကိုလည်း Apple Server မှာ သိမ်းဆည်းပေးထားတဲ့အတွက် Farook ရဲ့ iPhone 5C ကနေ iCloud backup လုပ်သမျှတွေကို FBI ကို ပေးခဲ့ပြီးဖြစ်ပါတယ်။ သို့သော် Backup က Apple server မှာ အောက်တိုဘာ ၁၉ ရက်နေ့အထိသာ ရှိပါတယ်။ ယခုလိုအခြေနေကြားထဲမှာ ပြည်နယ်ရဲ တစ်ဦးက Farook ရဲ့ Apple ID password ကို reset ချလိုက်မိတဲ့ အတွက် ခုဆိုရင် iPhone က Backup Server နဲ့ ချိပ်ဆက်မှုလည်း မရှိတော့ပါဘူး။

FBI ကတော့ ဘယ်လိုပဲ ဖြစ်နေပစေ အောက်တိုဘာ ၁၉ ရက် နောက်ပိုင်းကနေ ဒီဇင်ဘာ ၂ ရက်နေ့ထိ၊ အကြမ်းဖက်မှုတွေ ဖြစ်ပေါ်လာဖို့ အရေးကြီးတဲ့ ကျန်ရှိနေတဲ့ အချက်လက်တွေကို ရဖို့ customize software တစ်ခုကိုသာ တွင်တွင် ရေးပေးခိုင်းနေပါပြီ။

Apple ရဲ့ CEO ‘Tim Cook’ က “ဒီလို software မျိုးဟာ ဒီနေ့မှာ မရှိသေးပေမယ့် အကယ်၍ ရှိလာခဲ့ပြီး မသမာသူရဲ့ လက်ထဲကိုသာ ရောက်သွားခဲ့မယ်ဆိုရင် မည်သူရဲ့ iPhone ကို မဆို unlock လုပ်နိုင်သွားလိမ့်မယ်” လို့ ဆိုပါတယ်၊ ဆက်ပြီး “FBI အနေနဲ့တော့ ဒီ tool နဲ့ ပက်သက်ပြီး အခြားစကားတွေ ရှာပြောနေမှာပါပဲ။ ဒါပေမယ့် ဒီကိစ္စက အမှားမခံပါဘူး။ အခုလို လုံခြုံရေးကို ချိုးဖောက်ဖို့ ရည်ရွယ်ပြီး တီထွင်လိုက်တဲ့ iOS ဆိုတာဟာ နောက်တံခါးပေါက် (backdoor) တစ်ခု ဖောက်လိုက်တယ်ဆိုတာကိုတော့ ငြင်းလို့ မရနိုင်ပါဘူး၊ ပြီးတော့ အစိုးရအနေနဲ့ အခုချိန်မှာတော့ ဒီကိစ္စလေး တစ်ခုထဲ ဖြေရှင်းဖို့ပါ ဆိုပြီး စောဒကတက်နေပေမယ့် တစ်ကယ်တမ်းကြတော့ ထိန်းချုပ်နိုင်ဖို့ နည်းလမ်းဆိုတာ လုံးဝ မရှိပါဘူး”

ယခုတော့ Apple က အင်ဂျင်နီယာတွေက encryption tool အသစ်တစ်ခုကို ပြုလုပ်နေပါပြီ။ လက်ရှိစနစ်က user ရဲ့ iCloud Backup data နဲ့ Keys တွေကို Apple Server မှာ သိမ်းထားပေးတဲ့ အတွက် လိုအပ်ရင် Apple က data တွေကို decrypt (ပြန်ဖြည်) ပေးနိုင်ပါတယ်။ စနစ်အသစ်မှာတော့ iCloud Backup Data ကိုသာ server မှာ သိမ်းပေးတော့မှာ ဖြစ်ပြီး keys တွေကိုတော့ user device ရဲ့ local drive ထဲမှာပဲ မြုတ်နှံပေးတော့မှာဖြစ်ပါတယ်။ အဲ့ဒါကြောင့် user တွေအနေနဲ့ မိမိရဲ့ iCloud နဲ့ ပက်သက်တာတွေ၊ password reset ချဖို့ အချက်လက်တွေကို မေ့လျော့သွားခဲ့မယ်ဆိုရင်တော့ Backup data တွေ ဆုံးရံှုးမယ့် အလားအလာ ရှိနေပါတယ်။

တစ်ကယ်တော့ ဒီစနစ်ကို Apple က အင်ဂျင်နီယာတွေက ယခုလို အကြမ်းဖက်မှု မဖြစ်ခင်ထဲက စတင် အကောင်အထည်ဖော်နေခဲ့ကြတာပါ။

“အစိုးရရဲ့ အခုတောင်းဆိုချက်ကြီးက သိပ်ကြောက်စရာကောင်းပါတယ်၊ ဒါကိုသာ ဉပဒေပြုလိုက်မယ်ဆိုရင်၊ အဲဒီဥပဒေကို သုံးပြီး သင်တို့ရဲ့ iPhone တွေကို အလွယ်လေး unlock လုပ်နိုင်မယ်၊ ဒါဆိုရင် ဘယ်သူ့ရဲ့ device ထဲကပဲ ဖြစ်နေပစေ Data တွေကိုတောင် ယူလို့ရသွားမယ့် အနေအထားပါ။ အစိုးရအနေနဲ့ လူတစ်ယောက်ရဲ့ privacy ကို ချိုးဖောက်နိုင်တဲ့ လုပ်ဆောင်မှုတွေ တိုးကောင်း တိုးလာနိုင်မယ်၊ စောင့်ကြည့်နေနိုင်မယ့် software တွေ Apple ကို ရေးခိုင်းပြီး သင်တို့ရဲ့ message ၊ ကျန်းမာရေးမှတ်တမ်း၊ ငွေရေးကြေးရေး ကိစ္စတွေကို ကြားဖြတ်ထောက်လှမ်းရုံတင်မက၊ ဖုန်းကင်မရာ နဲ့ မိုက်ခရိုဖုန်းတို့ကိုပါ သင်တို့တောင်မသိလိုက်ပဲ ဝင်ရောက်ထောက်လှမ်းကောင်း ထောက်လှမ်းနိုင်ပါလိမ့်မယ်” လို့ customer letter မှာ Tim Cook ဖြည့်စွက်ရေးသားခဲ့ပါတယ်။

Apple အနေနဲ့ အနာဂါတ် လုံခြုံရေးနည်းပညာသစ် ပါလာမယ့် iOS device တွေမှာ Hardware design ပိုင်းကိုတောင် ပြောင်းလဲပစ်ဖို့ စဉ္းစားနေပါပြီ။ ဉပမာ firmware အသစ်တစ်ခုကို တင်တဲ့အချိန်မှာ passcode မှားယွင်းပါက system storage တစ်ခုလုံးကို ဖျက်ပစ်မည့် ပုံစံမျိုးဖြစ်နိုင်ပါတယ်။ လက်ရှိမှာတော့ firmware အသစ်တင်ရင် အဟောင်းပေါ်မှာ ထပ်ပြီး ရေးသွားတဲ့ စနစ်ဖြစ်ပြီး passcode မှန်ကန်ဖို့လည်း လိုအပ်ပါသေးတယ်။ firmware တင်ချိန်တွင် passcode မမှန်ကန်တာနဲ့ private user data တွေကို တန်းဖျက်ပစ်မည့် စနစ်က အစိုးရ အေဂျင်စီတွေ iPhone ကို hack လုပ်နိုင်မည့် အလားအလာ မရှိသလောက် နည်းသွားပြီး၊ Apple ရဲ့ assistance တောင် စွမ်းနိုင်တော့မည် မဟုတ်ပါ။

ယခု FBI နဲ့ အမှုမှာ အစိုးရဘက်က Backdoor နဲ့ ပက်သက်တဲ့ လုပ်ပိုင်ခွင် ဉပဒေတစ်ရပ်ကို ချမှတ်လိုက်ပါက နည်းပညာလောကကြီး တစ်ခုလုံးအတွက် အကြီးအကျယ် ဂယက်ရိုက်လာနိုင်ပါလိမ့်မည်။ လတ်တစ်လော Apple ရဲ့ဘက်မှာတော့ နည်းပညာကုမ္ပဏီကြီးများဖြစ်ကြတဲ့ Microsoft, Google, Facebook နဲ့ Twitter တို့ ပံ့ပိုးနေပြီး တရားရုံးတော်ကလည်း Apple ရဲ့ ဘက်မှာ ရှိနေပါစေလို့ မျှော်လင့်လျှက် ရှိနေကြပါတယ်…။
